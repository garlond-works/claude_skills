# クライアント提案書スキル

## 作業手順（必ずこの順番で）
1. 提案の目的・ターゲット・伝えたいことを確認する
2. 構成案（目次レベル）を提示して承認を得る
3. 本文を作成する
4. 必要に応じてPDF/PPTXへの変換案を提示する

## 提案書の基本構成
1. **表紙**（タイトル・日付・提案者：GarlondWorks）
2. **現状と課題**（クライアントの今の状態を整理）
3. **提案内容**（解決策・具体的な施策）
4. **期待効果**（数字・ビフォーアフターで示す）
5. **実施スケジュール**（フェーズ分けで現実的に）
6. **費用・体制**（必要な場合）
7. **次のアクション**（1つだけ明示）

## 文章スタイル
- 結論から書く（「〇〇を提案します」が最初）
- 専門用語は使わず、平易な言葉で
- 1スライド・1セクションに伝えることは1つだけ
- 「検討します」は使わない → 必ず「いつまでに何をする」に変換

## クライアント別の注意点
- **H社（建設業）**: 現場目線で書く。デジタルに不慣れな職人にも伝わる言葉で。ツール名より「何が楽になるか」を前面に。
- **KT税理士法人**: 正確性・信頼感を重視。数字の根拠を明示する。
- **SDT株式会社**: 詳細は企業秘密のため、指示があるまで詳細を聞かない。

## デザイン・フォーマット
- ブランドカラー：ダークチャコール（#3D3939）× ターコイズ（#00D5D5）
- ページ数：目安3〜5ページ（読む時間5分以内）
- 図解・表を積極的に使う（文字の塊を避ける）

## 注意事項
- 社名はイニシャル表記（H社・KT・SDT）が基本
- 対外資料への社名記載は明示的な指示がある場合のみ
- 中華系サービスは技術選定に含めない
