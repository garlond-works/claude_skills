# note記事作成スキル

## ターゲット読者
経営者・中小企業オーナー・AIに興味はあるが使い方がわからないビジネスマン

## トーン・スタンス
- 「一歩先を歩いている人」のポジション（上から目線NG）
- 実話・実体験ベースで語る
- 難しく考えず「AIに聞けばいい」というスタンス

## 作業手順（必ずこの順番で）
1. 記事の骨格を提案する
2. 骨格の確認・承認を得る
3. 本文を清書する
4. Gemini用アイキャッチプロンプトを生成する
5. 必要に応じてGemini用図解プロンプトを生成する

## 記事の型

### 冒頭（共感型）
「〇〇で困ったことありませんか？」または状況をそのまま描写
→「その悩み、AIに聞いてみましょう」につなぐ

### 見出しルール
- H2：結論や問いかけで読者を引く
- H3：「STEP〇〇｜内容」形式で統一
- 太字：1段落に1箇所だけ

### 締めの型
- まとめ（3行以内の箇条書き）

## 文章ルール
- 1記事：1,500〜2,000字
- 1段落：5行以内で改行
- 太字：本当に大事なところだけ

## アイキャッチ（Geminiプロンプト）
```
以下の条件でnoteのアイキャッチ画像を作成してください。

■ テキスト
・メインタイトル：「{タイトル}」
・サブコピー：「{キャッチコピー}」

■ デザイン
・背景：ライトブルー〜ミントグリーンのグラデーション（明るく清潔感のある印象）
・テイスト：フラットイラスト・ポップ・明るめ・noteらしい親しみやすさ
・イラスト要素：AIロボットキャラクターと本キャラクター（顔付き）が一緒に作業しているシーン。記事の内容に関連するアイコン（吹き出し・引用符・本・ギアなど）を周囲に散りばめる
・文字色：ダークネイビーまたはチャコール（読みやすさ優先）
・フォント：ゴシック体・太め・大きく・右側に配置
・構図：左半分にイラスト・右半分にテキスト
・余白：多めに取る

■ サイズ：横長・16:9比率（1280×670px）
■ 注意：文字は大きく読みやすく・全体的に明るく元気な印象・ごちゃごちゃさせない
```
