# デザインルール詳細 v2
# 参考：パワポ研（上場企業IR資料分析）・enpreth（43パターン解説）

---

## 1. デザインの大原則（まずここを守る）

プロ級のスライドは「足し算」ではなく「引き算」で作られる。
以下の6つを「しない」だけで見栄えが激変する。

| しないこと | 理由 |
|---|---|
| 1スライドに情報を詰め込まない | 余白がないとダサい。余白は「設計」 |
| スライドごとにレイアウトを変えない | 統一感がおしゃれの土台 |
| 4色以上使わない | 色は濃淡で変化させる。色相はズラさない |
| 明朝体を使わない | ゴシック体が読みやすさの基本 |
| アニメーションを多用しない | 細部のこだわりに時間を使う |
| パワポ標準の派手な図形を使わない | 基本図形を組み合わせてオリジナルを作る |

---

## 2. 余白・レイアウトの原則

### 基本グリッド（16:9 = 10" x 5.625"）

コンテンツエリアは端から最低0.4"、推奨0.5"のマージンを確保する。
タイトル帯は高さ0.68〜0.75"、その直下に0.04"のアクセントラインを引く。
コンテンツエリアの高さは4.0〜4.5"とし、下部にフッター用0.3"を確保する。

### 密度のルール

| 要素数 | 推奨レイアウト | 注意点 |
|---|---|---|
| 1メッセージ | 全幅カード or 大きな数字callout | 余白を最大限確保 |
| 2比較 | 左右2列（各4.45"幅） | 中央に0.1"ギャップ |
| 3要素 | 3列（各2.9〜3.0"幅） | フェーズ・カード・比較に最適 |
| 4以上 | スライドを分割する | 詰め込みは絶対NG |

1スライドに入れるテキストブロックは最大4つ。フォントサイズ最低11pt（プレゼン用は14〜18pt）。

### 情報の位置を統一する（必須）
スライドをまたいで「見出し」「本文」「フッター」の位置は固定する。
都度レイアウトを変えると、読者が何がどんな情報か判断できなくなる。

## 3. カラー使用ルール

### 配色の黄金比
Primary（ダーク系）を60〜70%、Accent（ターコイズ）を20〜30%、White/OffWhiteを残りに使う。

### ブランドカラーの濃淡展開（色相はズラさない）
メイン色から濃いバリアント、薄いバリアントの3段階で展開する。
たとえば #1E3A5F → #0A2540 → #4A7AB5 のように。
これだけで3色使えるように見えて実は1色系統。統一感の正体はこれ。

### タイトル・クロージングはダーク背景で「サンドイッチ構造」
スライド1（タイトル）とスライド末（クロージング）はダーク背景、
中間のコンテンツスライドはオフホワイト背景にする。
締まった印象とメリハリが生まれる。

### やってはいけない配色
4色以上のカードを1スライドに混在させない。
明度が近い色同士を隣接させない（コントラスト不足）。
全スライド同じ背景色にしない（変化がなく飽きる）。

## 4. フォント体系

### フォントサイズ（プレゼン資料基準）

スライドタイトル（ダーク背景）: 36〜44pt / Arial Black / bold / ターコイズ色
セクション見出し（タイトル帯）: 18〜22pt / Arial Black / bold / ホワイト
キャッチフレーズ             : 20〜24pt / Arial Black / bold / アクセント色
カードヘッダー               : 12〜15pt / Calibri / bold / ホワイト
本文・箇条書き               : 11〜13pt / Calibri / regular / ダーク
サブテキスト・注釈           : 10〜11pt / Calibri / regular / グレー

### ジャンプ率のルール
見出しと本文のフォントサイズ差は最低1.5倍を確保する。
見出し18pt / 本文12pt（1.5倍）はOK。
見出し16pt / 本文14pt（1.14倍）はNG（差が読み取れない）。

### 数値・英字はCalibriで統一
日本語フォントに英数字が混じると不格好になる。
pptxgenJSではfontFace: "Calibri"で英数字もきれいに処理される。

### 日本語フォントの使い分け
- **納品形態が `.pptx` のみ**（PowerPointで開く）→ `"Meiryo UI"` や `"Noto Sans JP"` を使ってよい。見栄えが大幅に改善する。
- **LibreOffice経由でPDF変換が必要**な場合 → Calibriに統一。日本語フォントが埋め込まれないため文字化けする。
- 迷ったら納品形態を確認してから決める。

### 単位は数値より小さいフォントサイズで表示する
数値を48pt前後にした場合、単位（円・%・件など）は16pt前後にする。
この差がおしゃれ感を生む。

## 5. おしゃれアイデア実装パターン集

パワポ研（上場IR資料分析）から抽出した秀逸なデザインアイデア。

### [A] 境界線で2エリアに分割（タイトルスライド）

左エリア（72%）をダーク背景、右エリア（28%）をさらに濃いダークにする。
境界にアクセント色の縦線（0.04"幅）を引く。
右エリアにアイコン3点を散りばめ、左にコピーを大きく配置する。
単純な全面ダーク背景より格段に洗練される。

```javascript
// 右エリア
s.addShape(pres.shapes.RECTANGLE, {
  x: 7.2, y: 0, w: 2.8, h: 5.625,
  fill: { color: DARK_ALT }, line: { color: DARK_ALT }
});
// 境界線
s.addShape(pres.shapes.RECTANGLE, {
  x: 7.2, y: 0, w: 0.04, h: 5.625,
  fill: { color: ACCENT }, line: { color: ACCENT }
});
```

### [B] タイトル帯の下部アクセントライン

タイトル帯（ダーク）の直下に、アクセント色の0.04"ラインを引く。
これだけでタイトル帯にプロフェッショナル感が出る。

```javascript
s.addShape(pres.shapes.RECTANGLE, {
  x: 0, y: TITLE_BAR_H, w: 10, h: 0.04,
  fill: { color: ACCENT }, line: { color: ACCENT }
});
```

### [C] 左サイドアクセントライン（ダーク背景スライド全般）

```javascript
s.addShape(pres.shapes.RECTANGLE, {
  x: 0, y: 0, w: 0.22, h: 5.625,
  fill: { color: ACCENT }, line: { color: ACCENT }
});
```

### [D] カードの影用底板技法

本物のshadowの代わりに、同色の半透明矩形を少しずらして敷く。
LibreOffice変換でも崩れない安定した影表現。

```javascript
// 底板（影として機能）
s.addShape(pres.shapes.RECTANGLE, {
  x: x + 0.07, y: y + 0.07, w: W, h: H,
  fill: { color: PHASE_COLOR, transparency: 88 },
  line: { color: PHASE_COLOR, pt: 0 }
});
// 本体カード
s.addShape(pres.shapes.RECTANGLE, {
  x, y, w: W, h: H,
  fill: { color: "FFFFFF" },
  line: { color: PHASE_COLOR, pt: 2.5 }
});
```

### [E] カードフッターノートタグ

カード下部に半透明の帯を敷いて問題点や注釈を入れる。

```javascript
s.addShape(pres.shapes.RECTANGLE, {
  x: cardX, y: cardY + cardH - 0.32, w: cardW, h: 0.32,
  fill: { color: CARD_COLOR, transparency: 82 },
  line: { color: CARD_COLOR, pt: 0 }
});
s.addText("問題: 手書き補正で不整合発生", {
  x: cardX + 0.1, y: cardY + cardH - 0.32,
  w: cardW - 0.2, h: 0.32,
  fontSize: 9.5, color: CARD_COLOR, italic: true,
  valign: "middle", fontFace: "Calibri", margin: 0
});
```

### [F] 区切り線でカード内を整理

```javascript
s.addShape(pres.shapes.LINE, {
  x: x + 0.14, y: sectionBreakY, w: cardW - 0.28, h: 0,
  line: { color: "E8EDF2", width: 0.8 }
});
```

### [G] ドット箇条書き（リストアイコン代替）

「・」の代わりに小さな正方形ドットで洗練させる。

```javascript
s.addShape(pres.shapes.RECTANGLE, {
  x: listX + 0.08, y: itemY + 0.19,
  w: 0.08, h: 0.08,
  fill: { color: CARD_COLOR }, line: { color: CARD_COLOR }
});
s.addText(itemText, {
  x: listX + 0.24, y: itemY, w: itemW, h: 0.5,
  fontSize: 11.5, color: "2D3748", fontFace: "Calibri"
});
```

### [H] 事業連動アイコン（react-iconsから最適選択）

スライドタイトルやカードヘッダーに、業種・テーマに連動したアイコンを入れる。
アイコンはカードヘッダー内の左端（テキストの前）に0.34〜0.44"で配置。

建設・現場 → FaHardHat
データ管理 → FaLayerGroup
自動化     → FaSyncAlt
経営・分析 → FaChartBar
契約・書類 → FaFileAlt
クラウド   → FaCloudUploadAlt
スマホ     → FaMobileAlt
ユーザー   → FaUsers

### [I] 大きな数字callout（KPI強調）

数値を48〜60ptで配置し、単位を小さく添える。
背景に薄いアクセント色の帯を敷いて強調する。

```javascript
s.addShape(pres.shapes.RECTANGLE, {
  x: 0.4, y: 1.2, w: 2.8, h: 1.6,
  fill: { color: ACCENT, transparency: 88 },
  line: { color: ACCENT, pt: 1.5 }
});
s.addText("98", {
  x: 0.5, y: 1.3, w: 2.0, h: 1.0,
  fontSize: 56, bold: true, color: ACCENT,
  fontFace: "Arial Black", margin: 0
});
s.addText("%", { x: 2.3, y: 1.8, w: 0.6, h: 0.5,
  fontSize: 18, color: ACCENT, fontFace: "Calibri", margin: 0 });
```

### [J] ブリッジスライド（セクション区切り）

長い提案書ではセクション間にブリッジスライドを挟む。
背景はPrimary色、中央に大きなセクション番号（ターコイズ）、
次に説明する内容を1行で予告するテキストを添える。

## 6. アーキテクチャ図のルール

### 階層コンテナ構造
外枠（ダーク系）の内側左端に0.12"のアクセントラインを引く。
ラベルテキストは左上に配置し、アイコンをその左に置く。
内側ボックスはやや明るいバリアント色で統一する。

### 矢印はLINEシェイプのみ使用
pres.shapes.ARROWはLibreOffice変換で崩れるため使わない。
LINE + dashType:"sysDash"で代用する。

```javascript
// 縦方向の接続線（破線）
s.addShape(pres.shapes.LINE, {
  x: centerX, y: y1, w: 0, h: 0.28,
  line: { color: ACCENT, width: 1.5, dashType: "sysDash" }
});
// レイヤー間接続（実線）
s.addShape(pres.shapes.LINE, {
  x: 5.0, y: layerBottom, w: 0, h: 0.22,
  line: { color: ACCENT, width: 1.5 }
});
```

外部サービスとの接続は破線（sysDash）、内部レイヤー間の接続は実線で区別する。

## 7. 日本語テキスト文字化け対策（必須）

### 根本原因
pptxgenJSは日本語フォントをembedしない。
LibreOffice変換時に日本語フォントが見つからないと文字化けする。

### 禁止事項
絵文字（✅ ❌ ⚠️ 等）は絶対に使用禁止。
全角記号と英数字の混在が最も危険。
日本語フォント指定（MSゴシック等）は行わない。

### 代替表現
チェックマーク → ヘッダー色をグリーンに変える
バツ → ヘッダー色をレッドに変える
警告 → 別カードや注釈テキストで表現
丸数字（①②③）は使用可能だが、絵文字との混在はNG

## 8. QAチェックリスト（毎回必須）

生成後にview toolで各スライドを目視確認する。

テキストがカード枠からはみ出していないか
絵文字・特殊文字が文字化けしていないか
隣接するカードが重なっていないか（最低0.1"ギャップ）
フォントサイズが最低11pt以上か
各スライドに十分な余白があるか（端から0.4"以上）
カラーが案件テーマと一致しているか
ジャンプ率が1.5倍以上確保されているか
アイコンのサイズが適切か（カード内: 0.34〜0.44"）
フッターにGarlondWorks署名があるか（クロージングスライド）
タイトル・クロージングはダーク背景になっているか

## 9. 仕上げチェック（おしゃれレベル確認）

上場IR資料レベルを目指すための最終確認。

サンドイッチ構造（ダーク→ライト→ダーク）になっているか
タイトル帯の下にアクセントライン（0.04"）が入っているか
カードにドット箇条書きまたはアイコンが入っているか
数値の単位は小さなフォントサイズになっているか
カードにノートタグ（半透明帯）で補足情報が入っているか
影用底板技法が使われているか（shadowより安定）
左サイドアクセントラインがダーク背景スライドにあるか
アイコンが業種・内容と連動しているか
境界線分割技法がタイトルスライドに使われているか
